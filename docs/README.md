# Public Documentation

This folder contains the public-facing documentation for the Breathe project -- a breath-controlled sailboat survival game built in Unity with custom alternative-control hardware.

These documents describe the project's concept, architecture intent, milestones, and hardware requirements at a level appropriate for portfolio review, academic evaluation, and general interest.

## Structure

- `_project-overview/`
  - `PROJECT_MASTER.md` -- project scope, milestone roadmap, and scope guardrails.
- `_pitch/`
  - `_Breathe_Pitch.md` -- public pitch narrative, design goals, and product direction.
- `mechanic-and-architecture/`
  - `CORE_MECHANIC_PLAN.md` -- breath-to-gameplay architecture, input abstraction, and quality targets.
- `hardware/`
  - `HARDWARE_OVERVIEW.md` -- vendor-agnostic hardware requirements and integration targets.

## What's Not Here

Implementation details, build instructions, procurement specifics, code snippets, and operational procedures are maintained privately. These public documents focus on what the project is and why it's built this way, not how to reproduce it.
